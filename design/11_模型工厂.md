# 模型工厂（Model Factory）

---

## 1. 概述

DeerFlow 的模型工厂通过配置驱动的方式，支持任意 LangChain 兼容的 LLM 提供者。用户只需在 `config.yaml` 中声明模型，无需修改代码即可切换或扩展模型。

```python
# packages/harness/deerflow/models/factory.py

def create_chat_model(
    name: str | None = None,
    thinking_enabled: bool = False,
    reasoning_effort: str | None = None,
    **kwargs,
) -> BaseChatModel:
    """从 config.yaml 创建 LLM 实例"""
    config = get_app_config()
    model_config = config.get_model_config(name)
    model_class = resolve_class(model_config.use, BaseChatModel)
    # ... 构造参数
    return model_class(**model_settings)
```

## 3. 配置字段

### 3.1 通用字段

```yaml
models:
  - name: gpt-4                  # 内部标识（唯一）
    display_name: GPT-4          # 展示名称
    use: langchain_openai:ChatOpenAI  # LangChain 类路径
    model: gpt-4                 # API 模型名
    api_key: $OPENAI_API_KEY     # 环境变量
    base_url: https://api.openai.com/v1  # API 端点（可选）
    request_timeout: 600.0
    max_retries: 2
    max_tokens: 4096
    temperature: 0.7
    supports_thinking: false
    supports_vision: true
```

### 3.2 Thinking 模型配置

```yaml
models:
  - name: deepseek-v3
    use: langchain_deepseek:ChatDeepSeek
    model: deepseek-chat
    supports_thinking: true
    when_thinking_enabled:
      extra_body:
        thinking:
          type: enabled
```

### 3.3 OpenAI Responses API

```yaml
models:
  - name: gpt-5
    use: langchain_openai:ChatOpenAI
    model: gpt-5
    use_responses_api: true
    output_version: responses/v1
```

### 3.4 vLLM 模型

```yaml
models:
  - name: qwen3-32b
    use: deerflow.models.vllm_provider:VllmChatModel
    model: Qwen/Qwen3-32B
    base_url: http://localhost:8000/v1
    supports_thinking: true
    when_thinking_enabled:
      extra_body:
        chat_template_kwargs:
          enable_thinking: true
```

## 4. Thinking 模式

### 4.1 启用流程

```python
def create_chat_model(name, thinking_enabled, **kwargs):
    model_config = config.get_model_config(name)

    if thinking_enabled:
        # 检查模型是否支持
        if not model_config.supports_thinking:
            logger.warning(f"Model {name} does not support thinking")

        # 应用 when_thinking_enabled 覆盖
        if model_config.when_thinking_enabled:
            model_settings = deep_merge(model_settings, model_config.when_thinking_enabled)
```

### 4.2 reasoning_effort

部分模型（如 Claude）支持 `reasoning_effort` 参数：

```python
# 通过 kwargs 传入
create_chat_model(name, thinking_enabled=True, reasoning_effort="high")
```

## 5. 反射系统

DeerFlow 使用动态类加载（反射）实例化模型：

```python
# packages/harness/deerflow/reflection/

def resolve_class(path: str, base_class=None):
    """将 'module.path:ClassName' 解析为 Python 类"""
    module_path, class_name = path.split(":")
    module = importlib.import_module(module_path)
    cls = getattr(module, class_name)
    if base_class and not issubclass(cls, base_class):
        raise TypeError(f"{cls} is not subclass of {base_class}")
    return cls
```

## 6. 自定义 Provider

DeerFlow 支持添加任意 LangChain 兼容的 Provider：

```yaml
# 1. 安装 LangChain 集成包
uv add langchain-{provider}

# 2. 在 config.yaml 中配置
models:
  - name: my-model
    use: langchain_{provider}:Chat{model}
    model: my-model-name
    api_key: $MY_API_KEY
    base_url: https://...
```
