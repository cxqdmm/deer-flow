# 项目概述与定位

---

## 1. 项目简介

**DeerFlow**（**D**eep **E**xploration and **E**fficient **R**esearch **Flow**）是一个开源的 **Super Agent Harness**，由字节跳动开发并维护。

### 1.1 什么是 Super Agent Harness

传统的 AI Agent 通常只能执行单一对话。DeerFlow 则将 Agent 需要的所有基础设施打包在一起：

- **执行环境**：沙箱隔离的文件系统和命令执行
- **长期记忆**：跨会话积累用户偏好和知识
- **技能系统**：可插拔的工作流模块（Skills）
- **子 Agent 编排**：并行任务分解与执行
- **工具生态**：内置 + MCP + 社区扩展

DeerFlow 不只是一个研究工具，它是一个**真正的 Agent 运行时**，让 Agent 可以完成从几分钟到几小时的复杂任务。

### 1.2 版本历史

| 版本 | 时间 | 说明 |
|------|------|------|
| v1.x | 2024 | Deep Research 框架，基于 LangChain |
| **v2.0** | 2026-02 | 彻底重写，从框架升级为 Super Agent Harness |

> DeerFlow 2.0 与 v1.x 无共用代码，v1.x 在 `main-1.x` 分支继续维护。

## 2. 核心能力

### 2.1 Skills（技能）

Skills 是 DeerFlow 能完成"几乎任何事"的核心。每个 Skill 是一个 Markdown 文件，定义了特定领域的工作流、最佳实践和参考资源。

```
skills/
├── public/               # 内置技能（随仓库发布）
│   ├── deep-research/    # 深度研究
│   ├── ppt-generation/   # PPT 生成
│   ├── image-generation/ # 图像生成
│   └── ...
└── custom/              # 自定义技能（gitignored）
```

DeerFlow 自带 20+ 内置 Skills，覆盖研究、文档生成、数据分析、前端设计、图像视频生成等多个领域。

### 2.2 Tools（工具）

DeerFlow 的工具系统分为四层：

| 层级 | 工具数 | 示例 |
|------|--------|------|
| **沙箱工具** | 7 | bash、ls、read/write/glob/grep/str_replace |
| **内置工具** | 5+ | present_files、task、ask_clarification、view_image |
| **社区工具** | 3+ | web_search、web_fetch、image_search（Tavily/Jina/Firecrawl） |
| **MCP 工具** | 动态 | 任何 MCP 协议兼容服务器 |

### 2.3 Sub-Agents（子 Agent）

复杂任务分解并行执行：

```
Lead Agent → 分解任务
           → 并行拉起多个 Sub-Agent（最多 3 个）
           → 汇总结果
```

每个 Sub-Agent 有独立上下文，不干扰主会话。

### 2.4 Sandbox（沙箱）

每个会话运行在隔离环境中：

- **LocalSandbox**：直接文件系统访问（开发用）
- **AioSandbox**：Docker 容器隔离（生产推荐）
- **Provisioner**：Kubernetes Pod 隔离（大规模部署）

### 2.5 Memory（记忆）

跨会话持久化用户信息：

- 工作背景、个人偏好、关注话题
- 置信度评分的事实库
- 自动提取 + 防抖更新

### 2.6 IM Channels（IM 集成）

直接通过即时通讯应用与 DeerFlow 对话：

- Telegram（简单，长轮询）
- Slack（中等，Socket Mode）
- 飞书/Lark（中等，WebSocket）
- 企业微信（中等，WebSocket）

## 3. 技术特点

### 3.1 模型无关

DeerFlow 对模型无强绑定，支持任何 OpenAI 兼容 API 的 LLM：

- OpenAI GPT 系列
- Anthropic Claude（原生 SDK 或 OpenAI 兼容）
- Google Gemini
- DeepSeek
- 火山引擎 Doubao
- MiniMax
- 本地 vLLM
- Codex CLI
- Claude Code OAuth

推荐使用支持长上下文（100k+）、推理能力、多模态输入的模型。

### 3.2 双运行时模式

| 模式 | 进程数 | 说明 |
|------|--------|------|
| 标准模式 | 4 | LangGraph Server 独立进程 |
| Gateway 模式 | 3 | Agent Runtime 嵌入 Gateway |

### 3.3 可观测性

内置 LangSmith 和 Langfuse 集成，所有 LLM 调用、Agent 执行、工具调用全程追踪。

### 3.4 安全设计

- 本地部署为默认（仅 127.0.0.1 访问）
- 沙箱隔离执行环境
- 工具调用可选 Guardrail 权限检查
- 路径遍历防护

## 5. 适用场景

- **深度研究**：自动搜索、分析、生成报告
- **数据分析**：多步骤数据处理与可视化
- **内容创作**：PPT、图像、视频、播客生成
- **代码助手**：代码审查、文档生成、bug 修复
- **个人助理**：跨会话记忆，持久化工作流

## 6. 快速体验

```bash
# 一行命令让 Coding Agent 帮你安装
"Help me clone DeerFlow if needed, then bootstrap it for local development"
```

或手动：

```bash
git clone https://github.com/bytedance/deer-flow.git
cd deer-flow
make config        # 生成本地配置
# 编辑 config.yaml 配置模型
make dev           # 启动服务
# 访问 http://localhost:2026
```
