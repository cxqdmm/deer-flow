# 安全防护（Guardrails）

---

## 1. 概述

DeerFlow 的安全防护通过 **GuardrailMiddleware** 实现，在每个工具调用执行前进行权限检查。防护是可选的，由 `config.yaml` 中的 `guardrails` 配置启用。

```
Agent LLM 生成 tool_calls
    │
    ▼
GuardrailMiddleware.evaluate(tool_call)
    │
    ├─ Allow → 继续执行
    └─ Deny → 返回错误 ToolMessage，不执行
```

---

## 2. Guardrail 提供者

DeerFlow 支持三种 Guardrail Provider：

| 提供者 | 说明 |
|--------|------|
| **AllowlistProvider** | 白名单模式（零依赖，内置） |
| **OAP Provider** | 外部 OAP 兼容服务 |
| **自定义 Provider** | 用户实现的 Provider |

### 2.1 AllowlistProvider

最简单的内置模式，通过配置指定允许/禁止的工具：

```yaml
guardrails:
  enabled: true
  provider:
    use: deerflow.guardrails.allowlist_provider:AllowlistGuardrailProvider
    config:
      denied_tools: ["bash", "write_file"]  # 禁止这些工具
      # 或
      allowed_tools: ["web_search", "web_fetch"]  # 仅允许这些
```

### 2.2 OAP Provider

连接外部 Aport Agent Guardrails 服务：

```yaml
guardrails:
  enabled: true
  provider:
    use: aport-agent-guardrails:GuardrailProvider
    config:
      api_url: https://guardrails.example.com
      api_key: $GUARDRAILS_API_KEY
      denied_tools: ["bash", "write_file"]
```

## 3. Provider 接口

```python
# packages/harness/deerflow/guardrails/base.py

class GuardrailProvider(ABC):
    @abstractmethod
    async def evaluate(
        self,
        tool_name: str,
        tool_args: dict,
        context: dict,
    ) -> GuardrailResult:
        """评估工具调用是否允许"""

@dataclass
class GuardrailResult:
    allowed: bool
    reason: str | None = None
```

---

## 4. GuardrailMiddleware

```python
# packages/harness/deerflow/agents/middlewares/guardrail_middleware.py

class GuardrailMiddleware:
    async def before_tool_call(self, tool, input, config):
        result = await self.provider.evaluate(
            tool_name=tool.name,
            tool_args=input,
            context=config.get("configurable", {}),
        )

        if not result.allowed:
            return ToolMessage(
                content=f"Tool call denied by guardrails: {result.reason}",
                tool_call_id=...,  # 从 config 注入
            )
```

---

## 6. 安全使用建议

DeerFlow 默认部署在本地（仅 127.0.0.1 访问）。如需跨网络部署：

- **IP 白名单**：使用防火墙限制访问来源
- **前置认证**：Nginx 配置强认证
- **网络隔离**：VLAN 隔离
- **Guardrails**：启用工具调用权限检查
