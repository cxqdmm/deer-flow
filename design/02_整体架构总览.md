# 整体架构总览

---

## 1. 系统定位

DeerFlow（**D**eep **E**xploration and **E**fficient **R**esearch **Flow**）是一个开源的 **Super Agent Harness**，基于 LangGraph + LangChain 构建。它将 sub-agents、memory、sandbox 和 extensible skills 组织在一起，让 AI Agent 可以完成复杂的多步骤任务。

```
用户请求 → DeerFlow Agent → Sub-Agents / Tools / Skills / Memory
```

---

## 2. 技术栈

| 层级 | 技术 |
|------|------|
| Agent 框架 | LangGraph 1.0.6+ |
| LLM 抽象 | LangChain 1.2.3+ |
| REST API | FastAPI 0.115.0+ |
| 前端 | Next.js 16 + React 19 + Tailwind CSS 4 |
| 协议集成 | MCP (langchain-mcp-adapters) |
| 沙箱隔离 | LocalSandbox / AioSandbox (Docker) |
| 文档转换 | markitdown |
| 搜索工具 | tavily-python, firecrawl-py, Jina AI |

## 4. 双运行时模式

### 4.1 标准模式（Standard）

```
make dev
```

| 服务 | 端口 | 说明 |
|------|------|------|
| LangGraph Server | 2024 | Agent 运行时（独立进程） |
| Gateway API | 8001 | REST API |
| Frontend | 3000 | Next.js 开发服务器 |
| Nginx | 2026 | 反向代理 |
| **总计** | **4 进程** | |

### 4.2 Gateway 模式（Gateway Mode，实验性）

```
make dev-pro
```

| 服务 | 端口 | 说明 |
|------|------|------|
| Gateway API | 8001 | 包含嵌入式 Agent Runtime |
| Frontend | 3000 | Next.js 开发服务器 |
| Nginx | 2026 | 反向代理 |
| **总计** | **3 进程** | 无需 LangGraph Server |

Gateway 模式通过 `RunManager` + `run_agent()` + `StreamBridge` 将 Agent 运行时嵌入到 Gateway 进程中，自己管理并发。

## 5. 后端架构（Harness / App 分层）

DeerFlow 后端严格分为两层：

```
┌─────────────────────────────────────────────────────┐
│  App 层（app.*）— 应用代码，不对外发布               │
│  · FastAPI Gateway API                              │
│  · IM Channel 集成（Feishu/Slack/Telegram/WeCom）  │
└───────────────────────┬─────────────────────────────┘
                        │ App → Harness（允许）
                        ▼
┌─────────────────────────────────────────────────────┐
│  Harness 层（deerflow.*）— 可发布的 Agent 框架包     │
│  · Agent 系统（Lead Agent + Middlewares）           │
│  · 工具系统（Sandbox + Builtin + MCP + Community）  │
│  · 记忆系统（Memory Extraction + Queue）            │
│  · 子 Agent 系统（Subagent Executor + Registry）     │
│  · 模型工厂（Model Factory）                        │
│  · 配置系统（Config + Extensions）                  │
└─────────────────────────────────────────────────────┘
                        ▲
                        │ 注意：Harness 层永远不能引用 App 层
```

### 5.1 Harness 模块（`packages/harness/deerflow/`）

| 模块 | 职责 |
|------|------|
| `agents/` | Lead Agent、Middleware 链、Memory、ThreadState |
| `sandbox/` | 沙箱执行（Local + AioSandbox） |
| `tools/` | 工具系统（内置工具、工具注册） |
| `subagents/` | 子 Agent 委托与执行 |
| `mcp/` | MCP 协议集成 |
| `skills/` | 技能发现与加载 |
| `models/` | 模型工厂（支持 thinking/vision） |
| `config/` | 配置系统 |
| `community/` | 社区工具（Tavily、Jina、Firecrawl 等） |
| `reflection/` | 动态模块加载（resolve_variable/resolve_class） |
| `client.py` | 嵌入式 Python 客户端（DeerFlowClient） |

### 5.2 App 模块（`app/`）

| 模块 | 职责 |
|------|------|
| `gateway/` | FastAPI 应用与路由 |
| `channels/` | IM 平台集成（Feishu/Slack/Telegram/WeCom） |

## 7. 线程隔离（Thread Isolation）

每个对话（Thread）拥有完全独立的资源：

```
.deer-flow/
└── threads/
    └── {thread_id}/
        ├── user-data/
        │   ├── workspace/     # Agent 工作目录
        │   ├── uploads/       # 上传文件
        │   └── outputs/       # 最终交付物
        └── acp-workspace/     # ACP Agent 工作目录
```

沙箱内 Agent 看到的虚拟路径映射：

| 虚拟路径 | 说明 |
|---------|------|
| `/mnt/user-data/workspace` | Agent 工作目录 |
| `/mnt/user-data/uploads` | 上传文件目录 |
| `/mnt/user-data/outputs` | 输出文件目录 |
| `/mnt/skills` | 技能目录（只读） |
| `/mnt/acp-workspace` | ACP Agent 工作目录（只读） |

## 9. 记忆系统（Memory）

```
用户对话
    │
    ▼
┌──────────────────────────────────────────────────────────┐
│  MemoryMiddleware                                         │
│  过滤消息（仅保留用户输入 + 最终 AI 响应）                  │
│  加入更新队列                                            │
└──────────────────────────────────────────────────────────┘
    │
    ▼（防抖，30s 默认）
┌──────────────────────────────────────────────────────────┐
│  MemoryQueue                                             │
│  per-thread 去重 + 批量处理                              │
└──────────────────────────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────────────────────────┐
│  MemoryUpdater（LLM 驱动）                               │
│  提取上下文更新 + 事实（fact）                             │
└──────────────────────────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────────────────────────┐
│  memory.json（持久化）                                    │
│  · userContext — 工作/个人/当前关注                      │
│  · history — 近期/早期/长期背景                           │
│  · facts — 置信度评分的事实                               │
└──────────────────────────────────────────────────────────┘
    │
    ▼（注入系统提示词）
┌──────────────────────────────────────────────────────────┐
│  Lead Agent                                              │
│  Top 15 facts + context → <memory> 标签                  │
└──────────────────────────────────────────────────────────┘
```

## 15. 关键数据流

### 15.1 文件上传流程

```
用户上传文件
    │
    ▼
POST /api/threads/{thread_id}/uploads
    │
    ▼
Gateway → 存储到 .deer-flow/threads/{thread_id}/user-data/uploads/
    │
    ├── PDF/PPT/Excel/Word → markitdown 转为 Markdown
    │
    ▼
UploadsMiddleware 注入文件列表
    │
    ▼
Agent 可通过 /mnt/user-data/uploads/ 访问
```

### 15.2 线程清理流程

```
前端删除对话
    │
    ▼
DELETE /api/langgraph/threads/{thread_id}（LangGraph）
    │
    ▼
DELETE /api/threads/{thread_id}（Gateway）
    │
    ▼
删除 .deer-flow/threads/{thread_id}/ 目录
```
