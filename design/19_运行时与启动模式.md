# 运行时模式与启动系统

---

## 1. 启动命令

| 命令 | 模式 | 进程数 | 说明 |
|------|------|--------|------|
| `make dev` | 标准开发 | 4 | LangGraph 独立进程，热更新 |
| `make dev-pro` | Gateway 开发 | 3 | Agent Runtime 嵌入 Gateway |
| `make docker-start` | Docker 开发 | — | Docker Compose，热更新 |
| `make up` | Docker 生产 | — | Docker Compose，预构建 |
| `make stop` | — | — | 停止所有服务 |

---

## 2. 脚本入口

```bash
./scripts/serve.sh [--dev|--prod] [--gateway] [--daemon] [--stop|--restart]
```

### 2.1 参数说明

| 参数 | 说明 |
|------|------|
| `--dev` | 开发模式（热更新），默认 |
| `--prod` | 生产模式（预构建） |
| `--gateway` | Gateway 模式（无 LangGraph Server） |
| `--daemon` | 后台运行（nohup） |
| `--stop` | 停止所有服务 |
| `--restart` | 重启（先停止后启动） |

---

## 3. 标准模式架构（make dev）

```
serve.sh --dev
    │
    ├─ LangGraph Server（uv run langgraph dev --no-browser）
    │       │
    │       └─ port 2024
    │
    ├─ Gateway API（uv run uvicorn app.gateway.app:app）
    │       │
    │       └─ port 8001
    │
    ├─ Frontend（pnpm dev）
    │       │
    │       └─ port 3000
    │
    └─ Nginx（反向代理）
            │
            └─ port 2026
```

---

## 4. Gateway 模式架构（make dev-pro）

```
serve.sh --dev --gateway
    │
    ├─ Gateway API + Agent Runtime
    │   ├─ FastAPI REST endpoints
    │   └─ StreamBridge + RunManager（嵌入式 Agent）
    │       │
    │       └─ port 8001
    │
    ├─ Frontend
    │       │
    │       └─ port 3000
    │
    └─ Nginx
            │
            └─ port 2026
```

### 4.1 嵌入式 Runtime

Gateway 模式使用 `StreamBridge` + `RunManager` 在同一进程中运行 Agent：

```python
# app/gateway/deps.py

async with langgraph_runtime(app):
    # StreamBridge: SSE 流式响应
    # RunManager: 异步任务调度
    # 无需独立 LangGraph Server
```

---

## 5. 服务端口

| 服务 | 标准模式端口 | Gateway 模式端口 |
|------|-------------|----------------|
| Nginx（入口） | 2026 | 2026 |
| Frontend | 3000 | 3000 |
| LangGraph | 2024 | —（嵌入 Gateway） |
| Gateway | 8001 | 8001 |

---

## 6. Nginx 路由

```nginx
server {
    listen 2026;

    location /api/langgraph/ {
        proxy_pass http://localhost:2024;  # 标准模式
        # 或 http://localhost:8001  # Gateway 模式
    }

    location /api/ {
        proxy_pass http://localhost:8001;
    }

    location / {
        proxy_pass http://localhost:3000;
    }
}
```

---

## 7. Docker 开发

```bash
make docker-init    # 拉取沙箱镜像（首次）
make docker-start   # 启动所有容器
make docker-stop   # 停止容器
```

Docker Compose 根据 `config.yaml` 的沙箱配置决定是否启动 `provisioner`。
