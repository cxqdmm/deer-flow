# 可观测性（LangSmith / Langfuse Tracing）

---

## 1. 概述

DeerFlow 内置对 LangSmith 和 Langfuse 的追踪支持。所有 LLM 调用、Agent 执行、工具调用全程可见。

```
Agent 运行
    │
    ▼
Tracing Callbacks
    │
    ├─ LangSmith Callback → LangSmith Dashboard
    └─ Langfuse Callback → Langfuse Dashboard
```

---

## 2. LangSmith 集成

### 2.1 配置

```bash
# .env
LANGSMITH_TRACING=true
LANGSMITH_ENDPOINT=https://api.smith.langchain.com
LANGSMITH_API_KEY=lsv2_pt_xxxxxxxxxxxxxxxx
LANGSMITH_PROJECT=my-project
```

### 2.2 追踪内容

LangSmith 追踪所有：

- LLM 请求/响应
- 中间件执行
- 工具调用
- 子 Agent 执行
- Token 使用

### 2.3 Run Metadata

DeerFlow 自动注入元数据用于追踪：

```python
config["metadata"] = {
    "agent_name": "lead_agent",
    "model_name": "gpt-4",
    "thinking_enabled": True,
    "subagent_enabled": False,
}
```

---

## 3. Langfuse 集成

### 3.1 配置

```bash
# .env
LANGFUSE_TRACING=true
LANGFUSE_PUBLIC_KEY=pk-lf-xxxxxxxxxxxxxxxx
LANGFUSE_SECRET_KEY=sk-lf-xxxxxxxxxxxxxxxx
LANGFUSE_BASE_URL=https://cloud.langfuse.com
```

### 3.2 自托管

```bash
LANGFUSE_BASE_URL=https://your-langfuse.example.com
```

---

## 4. 双提供者

如果同时启用 LangSmith 和 Langfuse，DeerFlow 会同时向两者报告：

```
Agent Run
    │
    ├─ LangSmith Callback ────→ LangSmith Dashboard
    └─ Langfuse Callback ────→ Langfuse Dashboard
```

---

## 5. 错误处理

如果追踪提供者被明确启用但凭证缺失，DeerFlow 在模型创建时直接报错（fail-fast），而不是静默禁用：

```python
if tracing_enabled and not credentials_present:
    raise ConfigurationError(
        f"Tracing provider '{provider}' is enabled but "
        f"required credentials are missing."
    )
```

---

## 6. Token 使用统计

可选的 Token 使用追踪：

```yaml
# config.yaml
token_usage:
  enabled: true
```

通过 `TokenUsageMiddleware` 记录每次 LLM 调用的：

- `input_tokens`
- `output_tokens`
- `total_tokens`
