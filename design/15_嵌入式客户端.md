# 嵌入式 Python 客户端（DeerFlowClient）

---

## 1. 概述

`DeerFlowClient` 允许将 DeerFlow 作为嵌入式 Python 库使用，无需启动 HTTP 服务。直接在进程中调用 Agent 和 Gateway 功能，返回值与 HTTP API 完全一致。

```python
from deerflow.client import DeerFlowClient

client = DeerFlowClient()

# 直接对话
response = client.chat("Analyze this paper", thread_id="my-thread")

# 流式响应
for event in client.stream("hello"):
    if event.type == "messages-tuple":
        print(event.data["content"])
```

---

## 2. Agent 对话

### 2.1 同步对话

```python
def chat(self, message: str, thread_id: str | None = None) -> str:
    """同步发送消息，返回最终文本响应"""
```

### 2.2 流式响应

```python
def stream(self, message: str, thread_id: str | None = None):
    """流式发送消息，yield SSE 事件"""
    # 事件类型：
    # "values" — 完整状态快照
    # "messages-tuple" — 逐消息更新
    # "end" — 流结束
```

## 3. Gateway 等价方法

| 方法 | 返回格式 | 等价 API |
|------|---------|---------|
| `list_models()` | `{"models": [...]}` | `GET /api/models` |
| `get_model(name)` | `{name, display_name, ...}` | `GET /api/models/{name}` |
| `get_mcp_config()` | `{"mcp_servers": {...}}` | `GET /api/mcp/config` |
| `update_mcp_config(servers)` | `{"mcp_servers": {...}}` | `PUT /api/mcp/config` |
| `list_skills()` | `{"skills": [...]}` | `GET /api/skills` |
| `get_skill(name)` | `{"name", description, ...}` | `GET /api/skills/{name}` |
| `update_skill(name, enabled)` | `{"name", enabled, ...}` | `PUT /api/skills/{name}` |
| `install_skill(path)` | `{"name", path, ...}` | `POST /api/skills/install` |
| `get_memory()` | dict | `GET /api/memory` |
| `reload_memory()` | dict | `POST /api/memory/reload` |
| `get_memory_config()` | dict | `GET /api/memory/config` |
| `get_memory_status()` | dict | `GET /api/memory/status` |
| `upload_files(thread_id, files)` | `{"success": True, "files": [...]}` | `POST /api/threads/{id}/uploads` |
| `list_uploads(thread_id)` | `{"files": [...], "count": N}` | `GET /api/threads/{id}/uploads/list` |
| `delete_upload(thread_id, filename)` | `{"success": True}` | `DELETE /api/threads/{id}/uploads/{filename}` |
| `get_artifact(thread_id, path)` | `(bytes, mime_type)` | `GET /api/threads/{id}/artifacts/{path}` |

---

## 4. 架构

```
DeerFlowClient
    │
    ├─ Agent 路径（不走 HTTP）
    │   ├─ create_agent()
    │   ├─ chat() → run_agent()
    │   └─ stream() → StreamBridge
    │
    └─ Gateway 路径
        └─ 直接调用 deerflow.* 模块
```

### 4.1 与 Gateway API 的区别

| 方面 | DeerFlowClient | Gateway API |
|------|---------------|-------------|
| 部署 | 无需 HTTP 服务 | 需要 FastAPI 服务 |
| 文件上传 | 接受 `Path` 对象 | 接受 `UploadFile` |
| 线程清理 | 无等价方法 | `DELETE /api/threads/{id}` |
| Artifacts | 返回 `(bytes, mime_type)` | 返回 `FileResponse` |

## 5. 使用示例

### 5.1 研究任务

```python
from deerflow.client import DeerFlowClient

client = DeerFlowClient(thread_id="research-001")

# 创建研究计划
response = client.chat("""
Research the latest developments in AI agents.
Focus on:
1. Architecture trends
2. Key players and products
3. Technical challenges
4. Future predictions
""")

print(response)
```

### 5.2 带流式响应的 UI

```python
client = DeerFlowClient()

for event in client.stream("Write a Python function to fibonacci"):
    if event.type == "messages-tuple" and event.data.get("type") == "ai":
        print(event.data["content"], end="", flush=True)
```

## 6. 配置

DeerFlowClient 自动读取与 HTTP 服务相同的配置文件：

```
~/.deer-flow/config.yaml
或 DEER_FLOW_CONFIG_PATH 环境变量指定的位置
```

## 7. 测试

Gateway 一致性测试（`TestGatewayConformance`）验证每个方法返回值与 Gateway Pydantic 响应模型一致：

```python
# backend/tests/test_client.py

class TestGatewayConformance:
    def test_list_models_conforms(self):
        response = client.list_models()
        # 解析为 Gateway 的 Pydantic 模型
        ModelsListResponse.model_validate(response)

    def test_list_skills_conforms(self):
        response = client.list_skills()
        SkillsListResponse.model_validate(response)
```
