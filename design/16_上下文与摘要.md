# 上下文工程（Summarization + Plan Mode）

---

## 1. 概述

DeerFlow 通过两个机制管理长对话的上下文：

| 机制 | 触发条件 | 目的 |
|------|---------|------|
| **SummarizationMiddleware** | Token 接近上限 | 压缩旧消息，减少上下文 |
| **Plan Mode** | `is_plan_mode=True` | 结构化任务跟踪 |

---

## 2. 上下文摘要（Summarization）

### 2.1 触发条件

```yaml
# config.yaml
summarization:
  enabled: true
  trigger:
    type: tokens              # tokens | messages | fraction
    value: 60000              # 60k tokens
  keep:
    type: last_n_messages    # last_n_messages | last_n_tokens
    value: 10
```

三种触发方式：

| 类型 | 说明 |
|------|------|
| `tokens` | 输入 token 总数超过阈值 |
| `messages` | 消息数量超过阈值 |
| `fraction` | 当前上下文 / 最大上下文 的比例 |

### 2.2 摘要策略

```python
# 触发后：
# 1. 保留最近 N 条消息（或 N tokens）
# 2. 其余消息调用 LLM 压缩为摘要
# 3. 摘要替换旧消息
```

### 2.3 摘要模型

```yaml
summarization:
  model_name: gpt-4o-mini  # 可指定轻量模型节省成本
```

## 3. Plan Mode（任务管理模式）

当 `is_plan_mode=True` 时启用，提供 `write_todos` 工具：

```python
# packages/harness/deerflow/agents/middlewares/todo_middleware.py

@tool("write_todos")
def write_todos_tool(
    todos: list[dict],  # [{"content": "...", "status": "in_progress"}]
):
    """创建和管理结构化任务列表"""
```

### 3.1 任务状态

| 状态 | 说明 |
|------|------|
| `pending` | 待处理 |
| `in_progress` | 进行中（同时最多 1 个） |
| `completed` | 已完成 |

### 3.2 使用规则

```
✓ 复杂任务（3+ 步骤）使用 TodoList
✓ 完成后立即标记 completed
✓ 始终保持 1 个 in_progress（非并行时）
✗ 简单任务直接完成，不使用 TodoList
✗ 批量完成（分开标记）
```

### 3.3 配置

运行时通过 `config.configurable` 启用：

```python
{
    "is_plan_mode": True,
}
```

## 5. TitleMiddleware（自动标题）

首轮交换后自动生成对话标题：

```python
# packages/harness/deerflow/agents/middlewares/title_middleware.py

class TitleMiddleware:
    def after_agent(self, messages, state, config):
        if state.get("title") is not None:
            return  # 已有标题，跳过

        title = await self._generate_title(messages)
        state["title"] = title
```

标题用于：
- 前端聊天列表展示
- 记忆提取的上下文
